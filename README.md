## ⚙️ What Each Folder Does & Why

### `data/` — The Datasets
Holds the synthetic CSV datasets generated by `main.py`. These are the inputs to both ML models. The data is **not real** — it's generated with NumPy to simulate realistic school patterns (normal students vs. anomalous ones, income brackets, sibling counts, etc.).

### `src/` — Data Generation Engine
Mohammed's code. `generator.py` creates synthetic student records. `features.py` converts those records into the specific numeric features each ML model expects. Run `main.py` to trigger this.

### `Om/` — Attendance Anomaly Detection
Om's machine learning module. Uses **Isolation Forest** (an unsupervised ML algorithm) to detect students whose attendance patterns are abnormal — specifically students who start with good attendance and then suddenly drop. The trained model is saved as a `.pkl` file.

### `Are_Samhith/` — Fee Default Prediction
Are Samhith's machine learning module. Uses **GradientBoostingClassifier** with a tuned threshold of 0.40 to predict whether a student will default on their next term's fees. Uses data like income bracket, sibling count, and past payment history. The model achieves ≥70% recall on the default class.

### `Jyothsna/` — REST API
Jyothsna's work — wraps both trained models (Om's + Are Samhith's) into a live web API using **FastAPI**. Anyone (a frontend, a dashboard, a mobile app) can send a student's data and get back a prediction instantly. Access the interactive docs at `http://localhost:8000/docs`.

## 🚀 How to Run — Step by Step (No Virtual Environment)

> **Prerequisites:** Python 3.9+ installed. All `.pkl` model files are already in the repo — no retraining needed.

### Step 1 — Open PowerShell and go to the project folder
```powershell
cd c:\Users\moham\Desktop\school_predictor
```

### Step 2 — Install all required packages (run once)
```powershell
pip install -r requirements.txt
```
This installs: `numpy`, `pandas`, `scikit-learn`, `fastapi`, `uvicorn`, `joblib`, `pydantic`

### Step 3 — (Optional) Regenerate datasets
Only needed if the `data/` CSV files are missing:
```powershell
python main.py
```

### Step 4 — (Optional) Retrain the models
Only needed if the `.pkl` files are missing. They are already trained and in the repo:
```powershell
# Attendance model (Om's)
python Om/notebooks/train_attendance_model.py

# Fee default model (Are Samhith's)
python Are_Samhith/models/train_v2_final.py
```

### Step 5 — Start the API server (Jyothsna's)
```powershell
uvicorn Jyothsna.main:app --reload
```
Expected output:
```
INFO:     Application startup complete.
INFO:     Uvicorn running on http://127.0.0.1:8000
```

### Step 6 — Open the interactive API in your browser
```
http://localhost:8000/docs
```

## 🧪 How to Test the API (Using the Browser)

Once the API is running and you open `http://localhost:8000/docs`, you'll see 3 endpoints:

### ① GET `/` — Health Check
Click → **Try it out** → **Execute**
Confirms the API is running and shows which models are loaded.

---

### ② POST `/predict/fee-default` — Will this student default on fees?

Click → **Try it out** → paste this in the **Request body** → **Execute**:

```json
{
  "student_id": "STU001",
  "current_term": 1,
  "income_encoded": 1,
  "transport_user": 1,
  "sibling_count": 2,
  "current_status": 0,
  "current_defaulted": 0,
  "is_low_income": 0,
  "has_many_siblings": 0,
  "was_late_or_worse": 0
}
```

**You will get back:**
```json
{
  "student_id": "STU001",
  "default_probability": 24.5,
  "risk_category": "LOW",
  "will_default_pred": 0
}
```

| Field | Meaning |
|-------|---------|
| `default_probability` | % chance this student defaults |
| `risk_category` | `LOW` / `MEDIUM` / `HIGH` |
| `will_default_pred` | `1` = will default, `0` = will not |

---

### ③ POST `/predict/attendance-anomaly` — Is this student's attendance abnormal?

Click → **Try it out** → paste this → **Execute**:

```json
{
  "student_id": "STU002",
  "attendance_rate": 0.45,
  "longest_absence_streak": 14,
  "absence_in_last_30_days": 10,
  "day_of_week_variance": 0.05
}
```

**You will get back:**
```json
{
  "student_id": "STU002",
  "predicted_anomaly": 1,
  "anomaly_score": 0.1823,
  "risk_level": "High"
}
```

| Field | Meaning |
|-------|---------|
| `predicted_anomaly` | `1` = anomalous, `0` = normal |
| `anomaly_score` | Higher = more irregular |
| `risk_level` | `Low` / `Medium` / `High` |

## 🔬 ML Models — Technical Details

### Attendance Anomaly Detection (Om)
- **Algorithm:** Isolation Forest (unsupervised)
- **Why Isolation Forest?** No labeled training data needed. It detects outliers by how easily a data point can be "isolated" from the rest.
- **Features used:** `attendance_rate`, `longest_absence_streak`, `absence_in_last_30_days`, `day_of_week_variance`
- **Target recall:** ≥ 60% on anomalous students
- **Anomaly pattern:** Students who start normal (90%+ attendance) then suddenly drop below 50%

### Fee Default Prediction (Are Samhith)
- **Algorithm:** GradientBoostingClassifier with tuned threshold (0.40)
- **Why GradientBoosting?** Handles class imbalance well, works with small tabular data, interpretable.
- **Features used:** `income_encoded`, `transport_user`, `sibling_count`, `current_status`, `current_defaulted`, `is_low_income`, `has_many_siblings`, `was_late_or_worse`, `current_term`
- **Target recall:** ≥ 70% on default class
- **No data leakage:** Term N features predict Term N+1 default only

## ⚠️ Troubleshooting

| Problem | Fix |
|---------|-----|
| `ModuleNotFoundError: fastapi` | Run `pip install -r requirements.txt` |
| `ModuleNotFoundError: Jyothsna` | Run uvicorn from the project root (`school_predictor/`), not from inside any subfolder |
| `503 model not loaded` | The `.pkl` file is missing — run the training script for that model |
| `422 Unprocessable Content` | Your JSON is missing a required field — use the exact JSON examples above |
| Port 8000 already in use | Run `uvicorn Jyothsna.main:app --reload --port 8001` and open `http://localhost:8001/docs` |
